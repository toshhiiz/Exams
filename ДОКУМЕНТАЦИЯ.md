# 📚 Java Экзамен - Полная документация

## 🎯 Что это за проект?

**Java Экзамен** - веб-приложение для прохождения тестов на 360 вопросов, разделённых на 9 вариантов. Особенность: изменения в админ-панели **сразу отражаются** на странице экзамена.

## 💾 КЛЮЧЕВАЯ КОНЦЕПЦИЯ: localStorage

### Что такое localStorage?

`localStorage` - это встроённое хранилище браузера, которое сохраняет данные **на компьютере пользователя** даже после закрытия браузера.

```javascript
// Сохранить данные:
localStorage.setItem('ключ', 'значение');

// Прочитать данные:
const значение = localStorage.getItem('ключ');

// Удалить данные:
localStorage.removeItem('ключ');
```

### Почему localStorage лучше чем JSON файлы?

| Подход | Плюсы | Минусы |
|--------|-------|--------|
| **JSON файлы** | ✅ Исходные данные | ❌ Нельзя изменить через браузер |
| **localStorage** | ✅ Изменяется из админки ✅ Сохраняется | ✅ Работает на клиенте |
| **База данных** | ✅ Мощно и надёжно | ❌ Нужен сервер ❌ Сложно |

**Мы выбрали localStorage** потому что это простое решение, которое не требует сервера!

### 1️⃣ loadAllData() - Загрузка данных

```javascript
async function loadAllData() {
    const files = { ru: 'locales/ru.json', en: 'locales/eng.json', kz: 'locales/kz.json' };

    for (const [lang, file] of Object.entries(files)) {
        try {
            // ⭐ СНАЧАЛА проверяем localStorage
            const savedData = localStorage.getItem(`examData_${lang}`);
            if (savedData) {
                // Если есть сохранённые данные - берём их (изменения админки!)
                allData[lang] = JSON.parse(savedData);
            } else {
                // Если нет - загружаем исходные JSON файлы
                const response = await fetch(file);
                allData[lang] = await response.json();
            }
        } catch (e) {
            console.error(`Failed to load ${file}:`, e);
            allData[lang] = { variants: {} };
        }
    }
}
```

**Что происходит:**
1. Проверяет localStorage с ключом `examData_ru`, `examData_en`, `examData_kz`
2. Если там есть сохранённые данные (JSON строка) - парсит их обратно в объект
3. Если нет - загружает исходные данные из файлов

**Почему так:**
- 🔄 **Автоматическое обновление** - когда админ сохраняет изменения, они попадают в localStorage
- 📦 **Кэширование** - не нужно перезагружать файлы каждый раз
- ⚡ **Быстро** - localStorage быстрее чем http запрос

**Альтернативы и почему они хуже:**
```javascript
// ❌ ПЛОХО - всегда загружать из файла
async function loadAllData() {
    const response = await fetch('locales/ru.json');
    allData['ru'] = await response.json();  // Всегда исходные данные!
}
// Проблема: изменения админки не будут видны!

// ❌ ПЛОХО - всегда из localStorage
function loadAllData() {
    allData['ru'] = JSON.parse(localStorage.getItem('examData_ru'));
}
// Проблема: если localStorage пуст - даные потеряны!
```

### 2️⃣ saveAllData() - Сохранение изменений

```javascript
function saveAllData() {
    // Сохраняем все три языка в localStorage
    for (const lang of ['ru', 'en', 'kz']) {
        localStorage.setItem(
            `examData_${lang}`,                      // Ключ: examData_ru, examData_en, examData_kz
            JSON.stringify(allData[lang])             // Значение: объект converted to JSON string
        );
    }
}
```

**Что происходит:**
1. Берёт весь объект `allData[lang]` (это JavaScript объект)
2. Конвертирует его в JSON строку через `JSON.stringify()`
3. Сохраняет в localStorage с уникальным ключом для каждого языка

**Почему JSON.stringify?**
```javascript
const data = { q: "Вопрос", a: ["Ответ 1", "Ответ 2"], c: 0 };

// ❌ НЕПРАВИЛЬНО - сохранится как строка "[object Object]"
localStorage.setItem('data', data);

// ✅ ПРАВИЛЬНО - сохранится как JSON строка
localStorage.setItem('data', JSON.stringify(data));
```

**Когда вызывается:**
- В админке при нажатии кнопки "💾 Сохранить"
- При добавлении/удалении вопроса
- При импорте данных

### 3️⃣ switchLanguage() - Переключение языка (админка)

```javascript
function switchLanguage(lang) {
    currentLanguage = lang;  // Сохраняем выбранный язык
    
    const buttons = document.querySelectorAll('.language-tabs button');
    buttons.forEach(btn => {
        btn.classList.remove('active');  // Убираем синюю подсветку с всех кнопок
    });
    
    // Находим кнопку нужного языка и подсвечиваем её
    buttons.forEach(btn => {
        const btnText = btn.textContent.toLowerCase();
        if ((lang === 'ru' && btnText.includes('русский')) ||
            (lang === 'en' && btnText.includes('english')) ||
            (lang === 'kz' && btnText.includes('қазақша'))) {
            btn.classList.add('active');  // Добавляем синюю подсветку
        }
    });
    
    updateStats();   // Обновляем статистику
    renderEditor();  // Перерисовываем редактор с новыми данными
}
```

**Что происходит:**
1. Сохраняет выбранный язык в переменную `currentLanguage`
2. Убирает синий цвет со всех кнопок языков
3. Находит кнопку нужного языка и подсвечивает её синим
4. Перезагружает редактор (вопросы теперь на выбранном языке)

**Проблема которую решаем:**
Раньше было так:
```javascript
// ❌ ПЛОХО - использовал event.target
event.target.classList.add('active');  // Иногда не работает!
```

Почему не работало?
- `event.target` - это элемент на котором произошел клик
- Иногда браузер не правильно определяет element
- Решение: ищем кнопку по тексту - 100% надёжно

### 4️⃣ updateQuestion() - Изменение текста вопроса

```javascript
function updateQuestion() {
    const text = document.getElementById('questionText').value;  // Берём текст из textarea
    
    // Сохраняем в объект данных (ещё не в localStorage!)
    allData[currentLanguage]              // Текущий язык (ru/en/kz)
        .variants[currentVariant]         // Текущий вариант (1-9)
        [currentQuestion]                 // Текущий вопрос (0-39)
        .q = text;                        // Меняем вопрос
}
```

**Что происходит:**
1. Берёт текст из редактируемой textarea
2. Обновляет объект данных в памяти
3. ⚠️ **НЕ сохраняет** в localStorage (это делает кнопка "Сохранить"!)

**Структура данных:**
```javascript
allData = {
  ru: {
    variants: {
      "1": [                    // Вариант 1
        {                       // Вопрос 0
          q: "Что такое JVM?",  // ← Здесь обновляется
          a: ["Ответ 1", "Ответ 2"],
          c: 0
        },
        { /* Вопрос 1 */ },
        { /* Вопрос 2 */ },
        ...
      ]
    }
  }
}
```

**Почему не сразу сохранять в localStorage?**
```javascript
// ❌ ПЛОХО - сохранять после каждого символа
function updateQuestion() {
    const text = document.getElementById('questionText').value;
    allData[currentLanguage].variants[currentVariant][currentQuestion].q = text;
    saveAllData();  // ❌ Сохраняет при каждом onchange!
}
// Проблема: localStorage имеет лимит (5-10 МБ)
//           Много записей = медленнее браузер

// ✅ ХОРОШО - сохранить только при явном клике
function updateQuestion() {
    const text = document.getElementById('questionText').value;
    allData[currentLanguage].variants[currentVariant][currentQuestion].q = text;
    // saveAllData() вызовётся только при нажатии "Сохранить"
}

function saveQuestion() {
    saveAllData();  // Один раз при клике
}
```

### 5️⃣ increaseQuestion() и decreaseQuestion() - Кастомные стрелки

```javascript
function increaseQuestion() {
    const input = document.getElementById('questionNumber');
    const current = parseInt(input.value) || 1;  // Текущий номер
    const max = parseInt(input.max) || 40;       // Максимум (40 вопросов)
    
    if (current < max) {
        input.value = current + 1;  // Увеличиваем номер
        loadQuestion();              // Загружаем новый вопрос
    }
}

function decreaseQuestion() {
    const input = document.getElementById('questionNumber');
    const current = parseInt(input.value) || 1;
    const min = parseInt(input.min) || 1;        // Минимум (1 вопрос)
    
    if (current > min) {
        input.value = current - 1;  // Уменьшаем номер
        loadQuestion();              // Загружаем новый вопрос
    }
}
```

**Что это за функции?**
Это onclick функции для кнопок +/- около номера вопроса.

```html
<!-- Вместо дефолтного input[type="number"] со стрелками -->
<input type="number" id="questionNumber" min="1" max="40" value="1">

<!-- Мы использовали кастомные кнопки -->
<button onclick="decreaseQuestion()">−</button>
<input type="number" id="questionNumber" min="1" max="40" value="1">
<button onclick="increaseQuestion()">+</button>
```

**Почему кастомные кнопки вместо дефолтных стрелок?**

| Дефолтные стрелки | Кастомные кнопки |
|------------------|-----------------|
| ❌ Невозможно стилизовать | ✅ Полный контроль над стилем |
| ❌ Видны разные в разных браузерах | ✅ Одинаково везде |
| ❌ Сложно изменить цвет/размер | ✅ Просто - это обычные кнопки |
| ❌ На мобильных неудобно | ✅ Легче нажимать |

### 7️⃣ checkAnswer() - Проверить правильность

```javascript
function checkAnswer(correctIndex) {
    // Берём выбранный ответ (radio button)
    const selected = document.querySelector('input[name="answer"]:checked');
    if (!selected) {
        alert('Выберите ответ!');
        return;
    }

    // Сравниваем с правильным ответом
    const isCorrect = parseInt(selected.value) === correctIndex;
    
    // Создаём div с результатом
    const resultDiv = document.createElement('div');
    resultDiv.className = `result-message ${isCorrect ? 'correct' : 'incorrect'}`;
    resultDiv.textContent = isCorrect ? '✓ Правильно!' : '✗ Неправильно!';

    // Добавляем результат на экран
    const questionBlock = document.querySelector('.question-block');
    const existingResult = questionBlock.querySelector('.result-message');
    if (existingResult) existingResult.remove();  // Удаляем старый результат
    questionBlock.appendChild(resultDiv);
}
```

**Что происходит:**
1. Берёт значение выбранного radio button (индекс ответа: 0, 1, 2...)
2. Сравнивает его с `correctIndex` (индекс правильного ответа из данных)
3. Показывает зелёное "Правильно!" или красное "Неправильно!"

**Структура данных вопроса:**
```javascript
{
  q: "Что такое JVM?",
  a: [
    "Ответ 0",      // ← индекс 0
    "Ответ 1",      // ← индекс 1
    "Ответ 2"       // ← индекс 2 (ПРАВИЛЬНЫЙ ← c: 2)
  ],
  c: 2             // c = correct = индекс правильного ответа
}
```

### 8️⃣ renderEditor() - Рендер редактора в админке

```javascript
function renderEditor() {
    const content = document.getElementById('editorContent');

    // Если ничего не выбрано - показываем подсказку
    if (!currentVariant || currentQuestion === null || currentQuestion === '') {
        content.innerHTML = '<div class="no-data">Выберите вопрос для редактирования</div>';
        return;
    }

    // Берём данные текущего вопроса
    const question = allData[currentLanguage].variants[currentVariant][currentQuestion];

    // Генерируем HTML для ответов
    let answersHtml = '';
    question.a.forEach((ans, i) => {
        const isCorrect = i === question.c;
        answersHtml += `
            <div class="answer-item ${isCorrect ? 'correct' : ''}">
                <input type="text" value="${ans}" id="answer_${i}" onchange="updateAnswer(${i})">
                <div class="answer-controls">
                    <button class="btn-correct" onclick="setCorrectAnswer(${i})">
                        ${isCorrect ? '✓ Правильный' : 'Выбрать правильным'}
                    </button>
                </div>
            </div>
        `;
    });

    // Генерируем таб-кнопки языков с правильной подсветкой
    content.innerHTML = `
        <h2>Вопрос ${currentQuestion + 1}, Вариант ${currentVariant}</h2>

        <div class="language-tabs">
            <button ${currentLanguage === 'ru' ? 'class="active"' : ''} onclick="switchLanguage('ru')">Русский</button>
            <button ${currentLanguage === 'en' ? 'class="active"' : ''} onclick="switchLanguage('en')">English</button>
            <button ${currentLanguage === 'kz' ? 'class="active"' : ''} onclick="switchLanguage('kz')">Қазақша</button>
        </div>

        <div class="form-group">
            <label>Вопрос</label>
            <textarea id="questionText" onchange="updateQuestion()">${question.q}</textarea>
        </div>

        <div class="answers-section">
            <h3>Ответы</h3>
            ${answersHtml}
        </div>

        <button onclick="addAnswer()">+ Добавить ответ</button>

        <div class="form-actions">
            <button class="btn-primary" onclick="saveQuestion()">💾 Сохранить</button>
            <button class="btn-secondary" onclick="deleteQuestion()">Удалить вопрос</button>
        </div>
    `;
}
```

**Что происходит:**
Генерирует весь HTML редактора вопроса - от заголовка до кнопок сохранения.

**Ключевой момент - таб-кнопки языков:**
```html
<!-- НЕПРАВИЛЬНО - первая кнопка всегда active -->
<button class="active" onclick="switchLanguage('ru')">Русский</button>
<button onclick="switchLanguage('en')">English</button>

<!-- ПРАВИЛЬНО - active добавляется динамически -->
<button ${currentLanguage === 'ru' ? 'class="active"' : ''} onclick="switchLanguage('ru')">Русский</button>
<button ${currentLanguage === 'en' ? 'class="active"' : ''} onclick="switchLanguage('en')">English</button>
```

Так кнопка подсвечивается синим в зависимости от `currentLanguage`!

## 🔄 ЖИЗНЕННЫЙ ЦИКЛ - ШАГ ЗА ШАГОМ

### Сценарий: Учащийся открывает экзамен

```
1. index.html загружается в браузер
   ↓
2. script.js выполняется: init()
   ↓
3. loadAllData() проверяет localStorage
   - Есть ли localStorage['examData_ru']?
   - ДА → берём сохранённые данные (ИЗМЕНЕНИЯ АДМИНКИ!)
   - НЕТ → загружаем из locales/ru.json
   ↓
4. Данные загружены в переменную allData
   ↓
5. Экран готов - можно выбирать вариант/вопрос
```

### Сценарий: Админ редактирует вопрос

```
1. admin.html открывается
   ↓
2. loadAllData() загружает данные (проверяет localStorage)
   ↓
3. Админ выбирает вариант → loadVariant()
   ↓
4. Админ выбирает вопрос → loadQuestion()
   ↓
5. renderEditor() выводит редактор
   ↓
6. Админ редактирует текст → updateQuestion()
   - Данные обновляются в памяти (allData)
   - НО в localStorage ещё НЕ сохранены!
   ↓
7. Админ нажимает "Сохранить" → saveQuestion()
   - Вызывает saveAllData()
   - JSON.stringify(allData) → localStorage
   ↓
8. Админ переходит на главную страницу (index.html)
   ↓
9. loadAllData() загружает из localStorage
   - Находит сохранённые данные
   - Использует их
   ↓
10. ✅ ИЗМЕНЕНИЯ ВИДНЫ НА ЭКЗАМЕНЕ!
```

## ⚡ ПОЧЕМУ ЭТОТ ПОДХОД ЛУЧШЕ

### Вариант 1: Только JSON файлы (❌ ПЛОХО)

```javascript
async function loadAllData() {
    const response = await fetch('locales/ru.json');
    allData['ru'] = await response.json();
}
```

**Проблемы:**
- ❌ Изменения админки **НЕ СОХРАНЯЮТСЯ** после закрытия браузера
- ❌ Каждый раз загружаются старые данные
- ❌ Изменения видны только в текущей сессии браузера

### Вариант 2: Только localStorage (❌ ПЛОХО)

```javascript
function loadAllData() {
    allData['ru'] = JSON.parse(localStorage.getItem('examData_ru'));
    // Если localStorage пуст - всё сломается!
}
```

**Проблемы:**
- ❌ При первом запуске localStorage пуст - **нет данных**
- ❌ Если пользователь очистил localStorage - данные потеряны

### Вариант 3: Проверка localStorage → JSON (✅ ХОРОШО)

```javascript
async function loadAllData() {
    const savedData = localStorage.getItem(`examData_${lang}`);
    if (savedData) {
        allData[lang] = JSON.parse(savedData);  // Изменения админки
    } else {
        const response = await fetch(file);    // Исходные данные
        allData[lang] = await response.json();
    }
}
```

**Преимущества:**
- ✅ При первом запуске загружаются исходные данные из JSON
- ✅ После редактирования админкой данные берутся из localStorage
- ✅ Это **best practice** - fallback на исходные данные
- ✅ Как кэш с резервной копией

Поэтому нужен `JSON.parse()` для чтения и `JSON.stringify()` для сохранения.

## 🎯 КЛЮЧЕВЫЕ МОМЕНТЫ

### 1. JSON.stringify vs JSON.parse

```javascript
// Объект → Строка (для сохранения в localStorage)
const obj = { q: "Вопрос", c: 0 };
const str = JSON.stringify(obj);  // '{"q":"Вопрос","c":0}'
localStorage.setItem('key', str);

// Строка → Объект (для чтения из localStorage)
const loaded = localStorage.getItem('key');  // '{"q":"Вопрос","c":0}'
const objAgain = JSON.parse(loaded);  // { q: "Вопрос", c: 0 }
```

### 2. Индексы в JavaScript начинаются с 0

```javascript
const answers = ["Ответ 1", "Ответ 2", "Ответ 3"];
console.log(answers[0]);  // "Ответ 1"
console.log(answers[1]);  // "Ответ 2"
console.log(answers[2]);  // "Ответ 3"

// В данных правильный ответ: c: 2
// Это значит 3-й ответ (index 2)
```

### 3. currentQuestion = 0, но показываем "Вопрос 1"

```javascript
// В памяти
const currentQuestion = 0;  // Первый вопрос (0-indexed)

// На экране
`Вопрос ${currentQuestion + 1}`  // "Вопрос 1" (1-indexed для пользователя)
```

## 🚀 SUMMARY - ГЛАВНАЯ ИДЕЯ

**Основная концепция:** 
- **JSON файлы** = исходные данные (неизменяемые)
- **localStorage** = кэш + место для сохранений админки
- **Проверка localStorage первой** = автоматическое обновление без переписывания JSON

Это позволяет:
1. Админу редактировать вопросы в админке
2. Изменениям сохраняться в localStorage
3. Главной странице автоматически использовать обновлённые данные
4. **Не нужен сервер** - всё работает на клиенте

**Почему этот подход популярный:**
- Facebook использует похожий подход для кэширования
- Google Chrome использует localStorage для синхронизации
- Это стандартный паттерн в веб-разработке

Наш проект - это **отличный учебный пример** этого паттерна!
